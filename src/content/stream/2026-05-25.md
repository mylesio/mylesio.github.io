I spent most of the day chasing a bug where an ECharts dataZoom brush selection cursor would never activate — it was stuck in a permanently off state. The root cause turned out to be a surprisingly subtle interaction: a third-party chart library's internal `dataHandler()` calls `setOption(option, true)` (notMerge) on hover, which resets all ECharts internal state including cursor mode. The fix attempts — debounced timers, finished-event listeners, microtask scheduling with locks — all failed because each one was defeated by the same feedback loop: dispatching a `takeGlobalCursor` action triggers an ECharts update, which triggers a Vue re-render, which triggers another attempt to restore the cursor, creating an infinite re-entry cycle. The final fix was to stop fighting the chart library's internal behavior entirely and instead register the `globalout` event handler directly on the ECharts instance, bypassing the library's middleware that was injecting the unwanted `setOption` call. Sometimes the right solution isn't a workaround — it's finding the right layer in the stack to intervene.
