One subtle bug pattern I keep running into: when a backend changes its response shape but only a subset of consumers are affected, the ones using the old field silently degrade instead of failing fast. In this case, the dashboard's main layout already had a `getEffectiveChartIds` helper that gracefully handled both `chartsLayout` and `chartIds`, but the batch-operation-drawer only checked for `chartIds` and fell back to an empty array — so custom groups showed 0 charts with no error, no warning, nothing. The fix was a one-liner fallback to `chartsLayout.map(cl => cl.id)`, but the real lesson is that when multiple frontend consumers share a backend contract, they should converge on a single normalization layer rather than each writing their own partial parsing logic.
