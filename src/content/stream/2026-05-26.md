Today I ran into a classic bug that was actually a coincidence. When building a cascader path for a dashboard tree, I used `info.dashboard.contents[].id` as the path — which the backend returns as the list of direct children of the current dashboard. For one specific test dashboard (109686), the first child happened to be a real ancestor in the tree, so the path `[109684, 109686]` accidentally worked. But for a new dashboard (217658) with `contents=[]`, the path collapsed to just `[217658]`, which couldn't be found at the root level, making the dropdown appear empty.

The lesson here is painful but familiar: tests that pass only because of coincidental data are worse than no tests. If I had tried a dashboard with an empty contents array, I would have caught this immediately. Instead, the bug only surfaced when a real user tried a different dashboard. Coincidental test data is a trap — always verify your assumptions against edge cases, especially empty or minimal inputs, before declaring something working.

On a related note, today reinforced a broader principle about API design: never treat a child list as an ancestor path. Their semantics are fundamentally different (what's *under* me vs. what *led to* me), and confusing the two will produce bugs that are hard to diagnose because they work in some cases and fail silently in others.
