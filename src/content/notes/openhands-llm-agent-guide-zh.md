OpenHands 是一个开源的自主软件开发平台，能够修改代码、运行命令、浏览网页、调用 API。源码：[github.com/All-Hands-AI/OpenHands](https://github.com/All-Hands-AI/OpenHands)

## 智能体流程

Agent 循环：接收任务 → LLM 推理 → 工具调用 → 观察结果 → 循环直到完成。

OpenHands 实现了清晰的职责分离：LLM 决定*做什么*，运行时*执行*，观察结果反馈回上下文。

## 多模型支持架构

OpenHands 使用 LiteLLM 作为统一适配层，同一套 Agent 代码无需修改即可运行在任意 LLM provider 上。核心洞见：将 Agent 逻辑与模型接口分离。

## 消息系统设计

**自定义 Message 类**——支持富内容格式，包括文本、图片和工具结果。两种序列化方式：
- 字符串序列化：用于纯文本内容
- 列表序列化：用于多媒体内容

**消息构建流程**——`_get_messages` 将事件历史转换为 LLM 可消费的消息格式。系统维护一个滚动窗口，并对静态前缀应用提示词缓存（对 Anthropic 模型尤为有效）。

## 对话记忆管理

`ConversationMemory` 处理 CodeAct Agent 的事件历史：

1. 运行时事件持续累积
2. 按需从事件构建消息
3. 对静态前缀应用提示词缓存
4. 长历史被摘要或截断以适应上下文窗口

设计原则：把记忆当做流水线，而不是日志堆积。

## 微代理（Microagents）系统

微代理是将领域知识或工作流指导注入 Agent 上下文的专用模块。三种类型：

**知识微代理**（`.openhands/microagents/knowledge/`）
- 通过关键词触发
- 包含领域专业知识：特定工具、语言、框架
- 示例："当遇到 Vue 2 时，注入响应式系统注意事项文档"

**任务微代理**（`.openhands/microagents/tasks/`）
- 提供分步骤的工作流引导
- 接受输入并适应不同场景
- 示例："当任务是 PR Review 时，走流程：checkout → 读 diff → 检查测试 → 总结"

**仓库微代理**
- 项目特定的上下文和约定
- 当 Agent 在已知代码库中工作时自动加载

## 核心洞见

1. **模型与逻辑解耦**——使用适配层（LiteLLM 或类似工具）。Agent 不应该关心运行在哪个模型上。

2. **记忆是流水线**——不要只是追加事件；有目的地构建消息，缓存稳定前缀，摘要可变的中间部分。

3. **微代理 = 上下文注入**——不是一个巨型系统提示词，而是基于任务类型的定向知识注入。更小更相关的上下文胜过面面俱到的上下文。

4. **观察 → 推理 → 行动循环**——Agent 从不"知道"发生了什么，只能看到它能观察到的内容。精心设计你的观察机制。

5. **文件系统即记忆**——长时间运行的 Agent 应该把状态外化到文件（如 TODO.md、DEV_LOG.md），而不是只依赖上下文内的历史记录。
