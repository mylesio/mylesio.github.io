## WorldMind 是什么？

一个自治智能系统，维护对某个领域的**活的心智模型**——实时观察变化、构建结构化记忆、运行多 Agent 推理、产出**可验证的预测**。

与分析仪表板展示"发生了什么"不同，WorldMind 告诉你"将会发生什么"——并追踪自己的预测是否正确。

### 核心循环

```
感知 → 记忆 → 推理 → 预测 → 验证 → 学习
  ↑                                    │
  └────────────────────────────────────┘
```

1. **感知**：持续从目标域摄入事件
2. **记忆**：在时序知识图谱中存储实体、关系和事件
3. **推理**：多个专业 Agent 分析不同维度
4. **预测**：综合 Agent 输出形成可验证的预测
5. **验证**：对照现实检查过往预测
6. **学习**：将准确率数据反馈给所有 Agent 进行校准

---

## 首个目标域：GitHub 开源生态

一个丰富的、可公开观察的、快速变化的世界：
- 清晰的实体（仓库、开发者、组织）
- 可观测的信号（star、fork、commit、issue）
- 可度量的结果（采用曲线、社区增长）
- 时序动态（趋势涌现、到达峰值、消退）

架构是领域无关的——同一流水线可适配媒体叙事、金融或物理环境。

## 五个 Agent

### 趋势 Agent
**使命**：第一个发现异常。

监控 star 速度/加速度、fork-star 比、贡献者画像、README 质量。统计异常检测 + LLM 定性评估。

关键启发式：首周 >100 stars/天是强早期信号。被已知"品味引领者"star 的仓库即使绝对数字低也值得关注。

### 网络 Agent
**使命**：理解谁重要、如何连接。

构建和维护开发者/项目关系图。计算中心性（PageRank、介数中心性），通过模块度检测社区，追踪跨社区的"桥梁"开发者。

### 技术 Agent
**使命**：追踪技术生命周期。

监控跨仓库的依赖图变化，识别迁移模式（"大量仓库正在去掉 A 库，加入 B 库"），将技术映射到生命周期阶段：萌芽 → 增长 → 成熟 → 衰退。

### 预测 Agent
**使命**：综合所有信号形成具体预测。

组合所有 Agent 的洞察。识别收敛信号（多个 Agent 指向同一方向 = 更高置信度）。应用来自历史准确率的基准率。生成具体的、可度量的、有时限的预测。

预测类别：增长 / 采用 / 社区 / 衰退 / 涌现。

### 质疑 Agent
**使命**：专业的怀疑者。

对每个预测：主动搜索反面证据，检查认知偏差（近因效应、幸存者偏差、炒作周期），验证证据是否真正支持结论，检查基准率，寻找混杂因素。

**质疑清单**：
1. 证据充分吗？
2. 时间框架现实吗？
3. 相反的情况可能发生吗？
4. 我们在对噪声做模式匹配吗？
5. 有更简单的解释吗？

## 记忆系统

WorldMind 的核心。没有持久化、结构化、时序索引的记忆，Agent 只是无状态的 LLM 调用。

### 实体库
每个实体（仓库、用户、组织）有结构化档案：元数据、时间线、关系、Agent 标注、版本历史。追加模式——事件被添加，很少修改。

### 事件日志
所有观测和 Agent 输出流经统一的 JSONL 事件日志。按日文件，内存索引最近 N 条，超过 90 天的事件归档摘要。

### 时序图
关系如何随时间演变。每条边有 `[validFrom, validTo?]` 时间戳。支持时间点查询、时序差分、路径查询和社区检测。

### 信念状态
Agent 对"世界现在如何"的集体理解。信念有置信度评分，无强化证据时衰减，每轮更新后重新生成。

## 设计原则

| 原则 | 实现 |
|------|------|
| 每个 Agent 一个职责 | 单一分析维度 |
| 事件驱动非轮询 | Agent 对事件做出反应 |
| 记忆支持 | Agent 记住过往分析及其准确率 |
| 自校准 | 预测准确率反馈到置信度 |
| 不确定性感知 | 信念有置信度评分，不是二值真假 |
| 可验证输出 | 每个预测有检查日期 |

---

## 长远愿景

同一架构适配任何可观测的领域。Agent 可以换，采集器可以换，但循环不变：感知 → 记忆 → 推理 → 预测 → 验证 → 学习。

终极目标：一个对复杂系统建立真正理解的系统——不只是模式匹配，而是能在人类观察者注意到之前预测涌现现象的因果模型。
