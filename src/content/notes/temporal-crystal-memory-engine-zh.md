## Temporal Crystal 是什么？

一个为 AI Agent 设计的记忆引擎，解决根本问题：**如何给无状态模型赋予持久化、可进化、自维护的记忆？**

答案：一条灵感来自生物睡眠记忆巩固机制的分层压缩流水线。

---

## 四层记忆结构

```
RAW → COMPRESSED → ABSTRACT → CRYSTAL
 │         │            │          │
 高容量    中等          低容量     极少
 │         │            │          │
 所有      关键事实      模式        不变的
 细节      保留          识别        真理
```

| 层级 | 存储内容 | 存活时间 | 示例 |
|------|---------|---------|------|
| **RAW** | 发生的一切 | 天–周 | "14:32 用户询问了 Vue 响应式" |
| **COMPRESSED** | 关键事实和决策 | 周–月 | "用户偏好 Composition API 而非 Options API" |
| **ABSTRACT** | 跨事件的模式 | 月 | "用户通过动手学习，而非读文档" |
| **CRYSTAL** | 不变的真理 | 永久 | "总是先展示代码示例，再解释理论" |

每层逐步变小且更精炼。LLM 自身驱动压缩——摘要是模型的天然甜区。

## 混合检索（v0.4）

三种检索策略通过 RRF（Reciprocal Rank Fusion，k=60）融合：

| 策略 | 优势 | 劣势 |
|------|------|------|
| **BM25**（中文 jieba 分词） | 精确词匹配，快 | 错过语义相似但词汇不同的 |
| **Embedding**（OpenAI 兼容，L2 归一化余弦） | 语义相似性 | 可能漏掉精确匹配，较慢 |
| **Graph**（BFS + hop 衰减） | 通过连接找到相关概念 | 依赖图的构建 |

融合确保同时获得精确度（BM25）和召回率（embedding），以及结构上下文（graph）。

## GraphRAG-Lite（v0.5）

轻量知识图谱，三种边类型：

| 边类型 | 含义 | 示例 |
|--------|------|------|
| `same_group` | 同一上下文的记忆 | 同一次调试 session 的两条记忆 |
| `semantic` | 语义相关内容 | "Vue 响应式" ↔ "基于 Proxy 的状态管理" |
| `call_chain` | 因果或程序性链接 | "构建失败" → "缺少依赖" → "npm install 修复了" |

检索通过 BFS 扩展：从最佳匹配节点开始，向外扩展并施加 hop 衰减（0.7^hop）。`call_chain` 边获得 +0.3 加分（因果链接比纯关联更有价值）。

## 自愈机制（v0.6）

当检索返回低质量结果（找不到相关内容）时，系统**生成新记忆来填补空白**：

```
查询 → 召回 → 质量太低？
                    ↓ 是
    LLM 合成 1-3 条新记忆
    写入 RAW 层
    用增强后的记忆库重新召回
```

约束：
- 每次查询最多 2 次自愈（防止失控生成）
- 自愈历史被追踪（可以审查合成了什么）
- `--dry-run` 模式：展示将生成什么，但不写入

## 梦境引擎（v0.7）

灵感来自生物睡眠巩固。每个梦境周期四个阶段：

### 阶段 1：DECAY（衰减）
应用 Ebbinghaus 遗忘曲线：`importance × e^(-rate × age_days)`

免疫规则：
- CRYSTAL 层记忆永不衰减
- 高重要度（≥0.8）记忆受保护
- 其余逐渐淡化

### 阶段 2：COMPRESS（压缩）
将存活过衰减的 RAW 记忆提升到 COMPRESSED 层。LLM 提取关键事实，丢弃过程性细节。

### 阶段 3：MERGE（合并）
通过 jieba token 重叠检测重复（Jaccard ≥ 阈值）。将相似记忆合并为单条更丰富的条目。

### 阶段 4：GRAPH（图更新）
用压缩和合并过程中发现的新连接更新知识图谱。

**Watch 模式**：作为后台线程按可配置间隔运行（像守护进程在你睡觉时做梦）。

## CLI 接口

```bash
tc init                    # 初始化新的 crystal 存储
tc add "记忆文本"           # 添加新记忆（自动评估重要度）
tc recall "查询"           # 检索相关记忆
tc recall --graph          # 包含图扩展结果
tc dream                   # 运行一个梦境周期
tc dream --watch           # 后台守护模式
tc dream --dry-run         # 预览不写入
tc heal "查询"             # 手动触发自愈
tc stats                   # 记忆层统计
tc graph-stats             # 知识图谱统计
```

- **可配置遗忘**：不同领域使用不同衰减曲线

## 核心洞见

传统 Agent 记忆要么"记住一切"（上下文溢出），要么"什么都不记"（无状态）。Temporal Crystal 提供第三种选择：**以正确的粒度记住正确的事，让其余的优雅地淡去**——就像生物记忆做的那样。

梦境引擎不是隐喻，它是睡眠巩固的字面实现：压缩、合并、遗忘、强化。结果是一个随时间变得*更好*而非只是更大的记忆系统。
