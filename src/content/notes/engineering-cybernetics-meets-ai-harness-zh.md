钱学森于 1954 年出版《工程控制论》。他将维纳的抽象控制论转化为可用于真实工程问题的方法论——火箭、导弹、自动化系统。核心思想：任何具有可测量输出和可控输入的系统，只要有反馈，就能让输出收敛到目标。

七十年后，同样的结构适用于让 AI Agent 可靠遵循指令并产出正确结果这一问题。

---

## 控制回路，重新表述

经典反馈控制图：

```
目标 ──▶ 控制器 ──▶ 执行器 ──▶ 被控系统 ──▶ 输出
           ▲                                  │
           └──────── 传感器（测量）◀────────────┘
```

换成 AI Agent：

```
Spec ──▶ Harness ──▶ Prompt ──▶ LLM Agent ──▶ 代码 / 文本
           ▲                                       │
           └──── Evidence（测试结果/lint/覆盖率）◀───┘
```

对应关系精确：

| 控制论概念 | AI Harness 对应 |
|-----------|----------------|
| 被控系统 | LLM Agent |
| 控制器 | 编排逻辑 + 规则 |
| 参考输入 | Spec + 验收标准 |
| 执行器 | Prompt 注入 |
| 传感器 | 自动化 Evidence 采集 |
| 反馈信号 | 测试运行器 / linter 的 pass/fail |
| 负反馈 | Fix-Loop（失败则重试） |

## 四层控制方法

钱学森描述了四种设计方法，从简单到复杂：

**L1 — 经典控制。** 系统模型已知，固定规则足够。对应 Harness：硬编码规则。"E2E 测试必须通过。""不允许硬编码密钥。"在 Agent 行为可预测时有效。

**L2 — 鲁棒控制。** 模型存在不确定性，控制器必须容忍一定范围的偏差。对应 Harness：Phase Gate——强制退出，不需要知道 Agent 为什么失败，只需知道输出超出可接受范围。

**L3 — 自适应控制。** 边控制边学习，控制器在线更新模型。对应 Harness：迭代 Benchmark + 复盘分析。每轮 Benchmark 暴露新的失败模式，提炼为下一轮的更新规则。控制器随时间改进。

**L4 — 智能控制。** 多 Agent 系统、目标冲突、涌现行为。对应 Harness：前沿方向——一个 Agent 的输出约束另一个、动态 Spec 漂移检测、多目标优化（覆盖率 vs token 消耗 vs 延迟）。

## 可观测性要求

控制论有一个基础定理：系统只有在**可观测**的情况下才能被控制——你必须能从输出推断系统的真实状态。

这对 AI Harness 设计有直接影响：**永远不要相信 Agent 的自我报告**。

一个说"我跑完测试了，全过了"的 Agent，不是传感器，它是被控系统的一部分。真正的传感器是测试运行器的输出文件——一个独立于 Agent 声明而存在的结构化产物。

Evidence-Based 验证意味着：把每个可测量的输出（测试结果、覆盖率数字、lint 报告、类型检查输出）当作事实真相，把 Agent 生成的文本当作未经验证的输入。Harness 只在 Artifacts 说"通过"时推进流程，而不是在 Agent 说的时候。

---

## 前馈 vs 反馈

大多数 Harness 工程是反馈主导的：Agent 产出 → 验证失败 → 重试。这是被动的。

控制论还有**前馈**：在执行前分析即将到来的任务，根据已知失败模式提前注入针对性约束。

实践中：如果任务涉及外部 HTTP 请求，在 spawn 时就注入约束"所有非 localhost 请求必须被拦截并 fulfill"，而不是等观察到第一次失败后再补。

前馈层通过将纠偏从被动转为预测，减少 Fix-Loop 次数。

---

## 稳定性

精心设计的反馈系统可能变得不稳定——振荡、发散、失控循环。

对应的 Harness 问题：无界 Fix-Loop。如果 Agent 反复未能通过同一个 Gate，而 Harness 无限重试，就会产生消耗资源却不收敛的自旋失败。

控制论的解决方案是在部署前分析闭环稳定性。Harness 的解决方案更简单：设置最大重试次数，超限则升级为人工审查。系统保持稳定，因为它有明确的失败模式（人在回路），而不是无穷递归。

---

## 这能给你什么

用控制论框架来思考 Harness 工程，能厘清真正需要构建什么：

1. **传感器优先。** 不可测量的就不可控。每个行为约束都需要一个对应的自动化度量。
2. **先鲁棒，再自适应。** 在投入学习机制之前，先让 Phase Gate 可靠运行。
3. **前馈减少反馈负担。** 在 spawn 时注入 100 条 Anti-Pattern，比 100 次 Fix-Loop 迭代成本更低。
4. **稳定性是设计约束，不是事后补丁。** 每个 Fix-Loop 都需要终止条件。

钱学森更深层的洞察是：**工程化**一个系统意味着让它的行为可预测、可控，而不只是祈祷它能用。同样适用于构建可靠的 AI Agent 流水线。
