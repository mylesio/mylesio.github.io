## 一、背景

2026 年的 Agent 工程视野中，核心目标已悄然从完成任务转向了持续学习。Manus 团队发布的上下文工程研究，以及全托管类 Agent 产品的上线，都或多或少涉及动态上下文、增量缓存、ralph loop 等概念。

共同线索：复用旧知识（MoE 动态参数、"一切皆文件"）来解决同一个问题——如何让 AI 通过长尾任务执行持续学习。

传统工作流是线性的。软件工程的实际运行逻辑并非线性，能运行的原因是局部的逻辑反复有人兜底。基于 Agent 的工作流像在混沌系统里自动驾驶：除了行为规则，更聪明的模型和高鲁棒性的上下文工程才是关键。

DeepSeek 在"更聪明的模型"上持续发力。两个关键架构创新：

- **mHC（流形约束超连接）**——降低梯度衰减/爆炸风险，模型深度可达 1000+ 层（对比推测 GPT-4/Gemini 1.5 约 100 层）。论文：arxiv.org/pdf/2512.24880
- **Engram（条件记忆模块）**——类似哈希表，将事实知识存储在 CPU 内存中，释放 MoE 算力专注推理。论文：arxiv.org/pdf/2601.07372

## 二、更聪明的模型

### 2.1 计算与记忆的解耦与融合

DeepSeek V4 的核心架构突破预计在于计算引擎与记忆引擎的物理分离。传统 Transformer 在处理知识密集型任务时，必须用宝贵的计算参数模拟检索过程，浪费算力。Engram 将事实性知识存储于低成本内存，让算力专注推理。

### 2.2 门控机制替代单一推理路线

上下文感知门控系统将 MoE 和 Engram 结合：
- 简单事实查询 → 直接激活记忆模块
- 复杂逻辑问题 → 关闭记忆通道，全量激活深度推理链路

实验证明：引入 Engram 后，即便只释放 40% MoE 专家席位，模型性能依然与纯 MoE 基线持平。

## 三、高鲁棒性的上下文工程

根据 Manus 团队发布的经验（《Context Engineering for AI Agents: Lessons from Building Manus》），即便"智能体摩尔定律"（每 7 个月任务完成时间跨度翻倍）持续成立，上下文工程依然是构建高效、低成本、稳定 Agent 的关键。

以"从零构建一个大盘页面"为例：

### 3.1 以 KV 缓存为核心的上下文工程

构建基于老旧技术栈（Vue 2 等）的 Agent 工作流时，建议把 KV 缓存命中率作为工程架构的第一指标。

如果每次交互都动态加载组件库文档、项目目录结构和工具定义，延迟和成本会爆炸。最佳实践：维护一套静态"系统前缀"——组件 API 参考、项目目录结构、基础工具定义——固定在上下文最前端，序列化顺序确定且稳定。

Agent 接到任务时，只需处理任务特定的增量信息。这样能确保缓存命中率维持在 90% 以上，即便面对复杂遗留代码库也能保持"秒回"。

### 3.2 文件系统即上下文总线

复杂项目有深层全局状态依赖（EventBus、Vuex）。单纯依赖内存中的对话历史容易导致 Agent 在长链路开发中"迷失中间"。

正确做法：授权 Agent 直接读写文件（`TODO.md`、`DEV_LOG.md`）。显式状态外化将模型注意力强行锚定在当前任务目标上。文件系统 = 无限容量外部存储，与 Engram 的参数化记忆形成互补：Engram 存储"Vue 2 响应式直觉"，文件存储"当前走到了哪一步"。

### 3.3 拥抱错误的沙盒演进机制

构建"宽容错误"的沙盒环境。当 Agent 生成的代码在沙盒中失败时，不要隐藏错误——把它当作学习契机。将完整错误堆栈保留在上下文中。让模型自我诊断（`Vue.set` vs. 直接索引修改）并自我修复。

将清洗后的沙盒运行日志积累成文件，逐步建立增量规则库。

### 3.4 基于 Agent Skills 的领域经验收敛链路

最后一块拼图：将零散修复系统化地收敛为领域知识。

示例：维护一个 `Vue2-Expert` 技能文档。当 Agent 在沙盒中修复了 `this.items[index]` 的问题后，自动触发"经验总结"——把修复抽象为设计模式，而非只是错误日志。将"数组响应式铁律"追加到技能文档。

防止无限膨胀：定期触发去重和压缩，将相似案例合并为高权重规则。生命周期：**沙盒试错 → 模式抽象 → 归并入库 → 定期剪枝**。

这就是技能文档成长为"资深老员工"的方式：持续积累，不污染上下文命名空间。

## 四、提示词工程的物理学本质

在 Transformer 架构视角下，提示词工程不再是玄学，而是高维空间中的精确制导。

**Attention 机制**决定了提示词是在构建查询向量。模糊指令产生发散向量。精准角色设定通过预设偏移量，在高维空间中精准击中特定知识簇。这解释了为什么角色扮演有效：它在模型层面物理排除了无关领域的干扰。

**FFN（前馈神经网络）**揭示了少样本学习的实质是特征激活。你提供的每个示例都在以数据共振激活模型内部神经元。逻辑一致性比数量更重要——高质量样本避免激活错误特征区域。

目标不是"命令"模型，而是在意义空间中设定导航起点并约束路径。
